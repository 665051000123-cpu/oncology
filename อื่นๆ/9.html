คำอธิบายฟีลด์: id (รหัสห้อง PK), room_name (ชื่อห้อง), location/floor (สถานที่/ชั้น), capacity (ความจุสูงสุด), equipment_summary (สรุปอุปกรณ์), status (สถานะการใช้งาน), created_at (วันที่สร้างข้อมูล) ตาราง bookings (ข้อมูลการจอง) SQL CREATE TABLE bookings ( id INT AUTO_INCREMENT PRIMARY KEY, room_id INT NOT NULL, user_id INT NOT NULL, title VARCHAR(200) NOT NULL, agenda TEXT, meeting_date DATE NOT NULL, start_time TIME NOT NULL, end_time TIME NOT NULL, attendee_count INT DEFAULT 0, approval_status ENUM('pending','approved','rejected') DEFAULT 'pending', booking_status ENUM('draft','confirmed','cancelled','completed') DEFAULT 'draft', created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, CONSTRAINT fk_booking_room FOREIGN KEY (room_id) REFERENCES rooms(id), CONSTRAINT fk_booking_user FOREIGN KEY (user_id) REFERENCES users(id) ); คำอธิบายฟีลด์เพิ่มเติม: room_id/user_id (FK อ้างอิงตารางห้องและผู้ใช้), title (หัวข้อประชุม), meeting_date (วันที่ประชุม), start_time/end_time (เวลาเริ่ม/สิ้นสุด), attendee_count (จำนวนผู้เข้าประชุม), approval_status (สถานะการอนุมัติ), booking_status (สถานะการจอง) แนวทางปฏิบัติที่ดี (Best Practices ด้านขวา): ใช้ Foreign Key (FK), ตั้ง Index ที่ meeting_date / room_id, กันเวลาซ้อนด้วย Business Logic, ใช้ ENUM สำหรับควบคุมค่าสถานะ, เก็บ created_at สำหรับสืบย้อนข้อมูล รูปภาพที่ 8: LINE_NOTE_260629_8.jpg (8. ตัวอย่าง Code หน้า Booking Form) ตัวอย่างหน้าจอฝั่งซ้ายแบบฟอร์มเพิ่มรายการจอง และโค้ดฝั่งขวา (PHP + HTML ร่วมกับ Bootstrap 5): ฟอร์มหน้าจอประกอบด้วย: ห้องประชุม (Dropdown), วันที่, เวลาเริ่ม, เวลาสิ้นสุด, หัวข้อประชุม, ผู้เข้าร่วม (คน), รายละเอียด และปุ่ม "บันทึก" สิ่งที่ไม่ควรพลาดในการเช็ก (ด้านซ้ายล่าง): ต้องทำ Validate เวลา, ป้องกัน CSRF, ตรวจสอบห้องว่างก่อนบันทึก, Sanitize input, แจ้ง error แบบเข้าใจง่าย ตัวอย่างโค้ดโครงสร้างฟอร์ม: เปิดแท็กฟอร์มชี้ไปที่ action="booking_store.php" เมธอด POST มีการใส่ Input ซ่อนสำหรับเก็บ เพื่อป้องกันภัยคุมคามแบบ CSRF การทำตัวเลือกห้องประชุมจะใช้ลูป foreach ($rooms as $room) ในการดึงข้อมูล room_name ออกมาแสดงในแท็ก
