10. Checklist ก่อนขึ้น Production

รายการตรวจสอบความพร้อมขั้นสุดท้ายก่อนนำระบบใช้งานจริงบนเซิร์ฟเวอร์โรงพยาบาล
01 Functional
  • ทดสอบการคำนวณ BSA
  • ตรวจสอบส่วนลดสูญเสียอวัยวะ
  • ตรวจสอบสูตร Calvert (AUC)
  • ตรวจสอบ Max GFR Cap
  • ทดสอบ Patient Check-in
  • ตรวจสอบการเลือกสูตรยาร่วม
  • จัดการค่ายาขีดจำกัดสูงสุด
  • บันทึกประวัติผลคำนวณจ่ายยา
02 Security
  • เปิดใช้งาน HTTPS เต็มรูปแบบ
  • มีระบบป้องกัน CSRF Token
  • แยกสิทธิ์ RBAC เภสัชกร/แอดมิน
  • แฮชรหัสผ่านด้วย BCrypt
  • ตรวจสอบและกรองค่า Clinical Input
  • ใช้ Prepared Statements ป้องกัน SQL
  • ตั้งเวลา Session Timeout ปลอดภัย
03 Database
  • ตั้งค่า Backup อัตโนมัติทุกคืน
  • ทดสอบการ Restore ข้อมูลสำเร็จ
  • ทำ Index ตารางสำคัญ (HN/ยา)
  • ตรวจสอบการเชื่อมความสัมพันธ์ FK
  • Timezone ตรงกับประเทศไทย
04 Performance
  • ทำ Pagination ตารางประวัติยา
  • ปรับแต่ง Query ค้นหาข้อมูลด่วน
  • บีบอัดไฟล์ Asset หน้าบ้าน (Minify)
  • ทำ Cache ข้อมูลสูตรยาและตัวยาร่วม
  • ลดขนาดรูปภาพโลโก้และสื่อในระบบ
05 Operations
  • ตรวจสอบ Cron job คำนวณอัตโนมัติ
  • ติดตั้งระบบ Log Monitoring ตรวจสอบสถานะ
  • ระบบ Error แจ้งเตือนข้อผิดพลาดหลัก
  • ตรวจสอบพื้นที่ดิสก์ (Disk Space) คนไข้
  • ตั้งค่าการแจ้งเตือน email และไลน์แอดมิน
06 Deployment
  • ตั้งค่าไฟล์ .env สภาพแวดล้อมจริง
  • ตั้งค่า APP_URL ตรงกับโดเมนหลัก
  • ตรวจสอบสิทธิ์เขียนโฟลเดอร์รายงานคนไข้
  • กำหนด File Permissions ให้ปลอดภัย
  • เปิดใช้งาน Production Mode สมบูรณ์
07 User Readiness
  • จัดทำไฟล์คู่มือการใช้งานสำหรับเภสัชกร
  • จัดอบรมการดูแลระบบให้กับทีมแอดมินไอที
  • ลงนามผ่านเกณฑ์การทดสอบระบบ (UAT Sign-off)
  • เตรียมข้อมูลช่องทางติดต่อด่วนฉุกเฉิน
ระบบพร้อมสำหรับ Go-Live เมื่อ:
ผ่านการทดสอบ UAT จากเภสัชกร
ผ่านรายงาน Security Review
ระบบสำรองข้อมูลทำงานสมบูรณ์
