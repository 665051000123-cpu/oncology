API Flow

ลำดับการทำงานของ API (ระบบคำนวณและตรวจสอบขนาดยาเคมีบำบัด)

API Endpoints (ในระบบ)

POST /api/auth/login
GET /api/patients
POST /api/patients
GET /api/drugs
POST /api/calculations
PUT /api/users/password
GET ดึงข้อมูลคนไข้/ตัวยา (Read)
POST บันทึกการคำนวณ/ผู้ใช้ (Create)
PUT แก้ไขข้อมูล/รหัสผ่าน (Update)
1. User Interface

หน้าเว็บระบบ

React SPA (src/App.jsx)

แพทย์/เภสัชกร กรอกสัดส่วนคนไข้
2. Route

กำหนดเส้นทาง

Express Endpoint (server.js)

ชี้เป้าหมายตาม URL
3. Middleware
JWT Authentication
Role Verification
Express JSON parser
CORS Security
สกัดและกรองสิทธิ์การเข้าถึง
4. Controller

รับ Request

ถอด req.body (น้ำหนัก, ส่วนสูง, รหัสยา)

รับค่าพารามิเตอร์ของแอป
5. Service Layer
คำนวณหาค่า BSA
คำนวณ Chemotherapy Dose
ตรวจสอบ Overdose
สอบทานความปลอดภัย
ลอจิกการคำนวณสูตรแพทย์
6. Model

Data Mapper

จัดโครงสร้าง SQL Query Object

เตรียมคำสั่งเขียน/อ่านตาราง
7. Database

MySQL DB

ตาราง patients, drugs, logs

INSERT/SELECT ข้อมูลเสร็จสิ้น

8. Alert & Warning System

แสดงป็อปอัปแจ้งเตือนบนหน้าจอทันทีหากระบบตรวจพบปริมาณยาเกินกำหนด (Overdose)

9. Audit Log (ประวัติการแก้ไข)

บันทึกข้อมูลการคำนวณและประวัติผู้ใช้งานที่เข้ามาปรับปรุงค่าปริมาณยาเคมี
